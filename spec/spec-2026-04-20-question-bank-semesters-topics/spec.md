# homework-tutor - 题库按人教版学期拆分与知识点标注

## 1. 应用概述

### 1.1 需求背景
- 当前题库存放于 `grade1.js` ~ `grade6.js`，按 `QB[年级][学科]` 组织，共 1580 道题。
- 现有每题都带 `topic` 字段，但仅是宽泛标签，还没有严格对应到人教版教材的上册/下册，也没有统一的教材知识点口径。
- 当前闯关逻辑、题目展示、错题库持久化都只识别 `topic`，没有“学期”和“知识点列表”维度。
- 本次需求需要基于人教版教材，将 1-6 年级现有题库按学期拆分，并为题目补齐可展示、可检索、可持久化的知识点元数据。

### 1.2 技术栈
- **前端**: 原生 HTML + CSS + JavaScript 单页应用（`index.html`）
- **后端**: Node.js 原生 `http` 服务（`proxy.js`）
- **数据库**: SQLite（`wrongbook.db`）
- **其他**: 静态题库脚本、`localStorage` 存档、`better-sqlite3`

### 1.3 核心功能
1. 将现有题库按“人教版 + 年级 + 学科 + 学期”建立标准化教材映射。
2. 为每题补齐 `semester` 与 `knowledgePoints` 元数据，并保留对现有 `topic`/闯关逻辑的兼容。
3. 在闯关页、错题库等链路展示学期和知识点信息，为后续精细化出题与学习分析打基础。

### 2.3 各模块改动详情

#### 题库装载层
- **新增教材映射文件**：维护“年级 + 学科 + topic/题目特征 -> 学期 + 标准知识点”的人教版映射表。
- **`questions.js`**：新增题库规范化与校验逻辑，在题库加载后自动为每题补齐元数据，并构建学期索引。
- **`grade1.js` ~ `grade6.js`**：尽量不大面积重写现有题干；仅对无法通过 topic 自动归类的题目增加精确覆写。

#### 前端展示层
- **`index.html`**：
  - 增加学期选择能力或学期过滤状态；
  - `startBattle()` 按学期从题库取题；
  - 题目头部、错题库等位置展示 `学科 · 学期 · 主知识点`；
  - 保持 AI 拍照题、旧数据题在缺失学期时仍可正常显示。

#### 后端持久化层
- **`proxy.js`**：
  - 扩展错题表结构，增加 `semester`、`knowledge_points`；
  - 查询与写入接口向前端透传新增字段；
  - 兼容旧库，采用幂等列补齐方式迁移。

## 3. 功能规范

### 3.1 题库教材元数据标准化

**需求描述**: 所有内置题目都要带上与人教版教材一致的学期和知识点标签。

**实现方式**:
1. 以“年级 + 学科 + 现有 topic”为主键建立教材映射表，给出标准 `semester` 与 `knowledgePoints`。
2. 对于 `加减法`、`应用题`、`句型`、`古诗` 这类过宽 topic，增加题目级覆写规则，避免错误归册。
3. 题库加载完成后统一执行 `annotateQuestionBank()`，补齐每道题的标准字段。

**标准字段**:
- `semester`: `upper` / `lower`
- `semesterLabel`: `上册` / `下册`
- `topic`: 标准主知识点名称
- `knowledgePoints`: 细粒度知识点数组，至少 1 项

**关键逻辑**:
- 优先使用题目级覆写，其次使用 topic 映射。
- 对未命中的题目输出校验清单，禁止静默漏标。
- 保留 `QB[g][subj]` 扁平数组，避免一次性推翻现有代码。

**边界条件**:
- 当前题库里存在部分泛化练习题，无法只靠 topic 精准归入教材册次，必须允许手工覆写。
- 三年级以上科学学科也要跟随年级教材拆分，不能只处理语数英。

### 3.2 学期拆分与闯关取题

**需求描述**: 闯关题目不能再只按“年级 + 学科 + 难度”取题，而要引入学期维度。

**实现方式**:
1. 在题库规范化时生成学期索引，例如 `QB_TERM_INDEX[grade][subject][semester]`。
2. 前端进入副本时增加学期状态，战斗开始后只从对应学期题池抽题。
3. 题目展示区显示 `学科 · 学期 · 主知识点`，错题库列表展示学期与知识点摘要。

**关键逻辑**:
- 学期题池为空时给出明确提示，而不是直接开始空战斗。
- 旧题目若因数据未完成暂缺 `semester`，前端要降级显示为“未标注”，不能报错。

**边界条件**:
- 当前通关进度 key 为 `${grade}-${subject}-${level}`，如果升级成学期独立闯关，进度 key 需要变更为 `${grade}-${subject}-${semester}-${level}`。
- 旧存档迁移存在产品取舍：
  - 方案 A：学期独立闯关，旧进度不自动拆分，后续学期关卡重新累计。
  - 方案 B：学期只做筛选标签，不拆分通关进度。
  - 推荐采用 **方案 A**，更贴合教材节奏，但需要用户确认是否接受旧进度不精确迁移。

### 3.3 错题库元数据透传

**需求描述**: 错题库需要保留学期与知识点，方便后续按册次复习。

**实现方式**:
1. 前端写入错题时带上 `semester`、`knowledgePoints`。
2. SQLite 表新增对应列并兼容旧数据。
3. 错题列表优先展示 `semesterLabel + topic`，缺失字段时回退到现有 `topic`。

**关键逻辑**:
- `knowledgePoints` 使用 JSON 字符串持久化。
- 旧记录没有新增字段时，接口返回空值而不是失败。

### 3.4 AI 拍照题兼容策略

**需求描述**: 本次重点是“内置题库”改造，但不能让拍照出题链路因为新增字段失效。

**实现方式**:
1. `normalizeQuestion()` 接受缺省的 `semester`/`knowledgePoints`。
2. 展示层对 AI 题采用“无学期时仅显示 topic”的兼容逻辑。
3. 本轮不强制要求 AI 输出教材学期，避免把非人教版作业图片误归册。

## 4. 数据模型

|--------|------|------|------|

## 5. API 接口规范

### 5.1 错题库接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/wrongbook?user=...` | 获取错题列表，返回新增的学期与知识点字段 |
| POST | `/api/wrongbook?user=...` | 新增错题，支持 `semester`、`knowledgePoints` |
| DELETE | `/api/wrongbook/<id>?user=...` | 删除单条错题 |
| DELETE | `/api/wrongbook?user=...` | 清空错题 |

**请求示例** (POST):
```json
{
  "q": "9 + 6 = ?",
  "answer": "15",
  "userAnswer": "14",
  "topic": "20以内进位加法",
  "semester": "upper",
  "knowledgePoints": ["20以内进位加法", "凑十法"]
}
```

**响应要求**:
- 旧记录允许 `semester`、`knowledgePoints` 为空。
- 新字段缺失时后端不报错，但前端新增写入必须尽量补齐。

## 6. 配置规范

### 6.1 题库组织方式

```text
questions.js                # 题库初始化 + annotateQuestionBank
curriculum.js               # 新增，人教版学期与知识点映射
grade1.js ~ grade6.js       # 原始题目定义，必要时补覆写
index.html                  # 学期选择 / 题目展示
proxy.js                    # 错题库字段扩展
```

### 6.2 映射规则约定
- 映射表以“教材优先”而不是“题目生成方式优先”。
- 同一个 topic 在不同年级、学科可映射到不同 semester。
- 对模糊题的覆写规则必须可追踪，不能散落在 UI 逻辑里。

---

## 7. 安全规范

### 7.1 数据安全
- 新增字段仅为教材元数据，不引入新的外部输入面。
- `knowledgePoints` 入库前按 JSON 序列化，避免拼接 SQL。

### 7.2 兼容性安全
- 旧错题库和旧本地存档必须可以继续读取。
- 题库映射失败的题目要在控制台或校验脚本中输出，不允许无提示丢失。

---

## 8. 性能规范

### 8.1 性能目标
- 题库元数据补齐发生在页面加载阶段，P95 不明显增加首屏等待。
- 战斗开始时学期筛题仍保持前端本地毫秒级响应。

### 8.2 优化措施
- 映射结果在加载时一次性写回题目对象，避免每次开战重复计算。
- 学期索引提前构建，减少 `startBattle()` 中的遍历开销。

- 验证旧错题记录、AI 拍照题、无学期题目的兼容展示。
