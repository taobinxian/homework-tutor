# homework-tutor - 任务清单

## 阶段一：题库教材映射设计（优先级: 高）

### 任务 1.1: 新增人教版学期与知识点映射表
**所属应用**: homework-tutor
**所属模块**: 静态题库脚本
**影响文件**:
- 新增: `curriculum.js`
- 新增: `spec/spec-2026-04-20-question-bank-semesters-topics/`（当前 SPEC 文档）

**描述**: 为 1-6 年级现有学科建立教材映射表，定义标准学期与知识点口径。

**详细步骤**:
1. 盘点现有 `topic` 清单与高频模糊 topic。
2. 按人教版教材建立 `grade + subject + topic -> semester + knowledgePoints` 映射。
3. 为模糊 topic 预留题目级覆写结构。

**验收标准**:
- 每个已有年级/学科都有教材映射。
- 映射表支持 `semester`、`semesterLabel`、`knowledgePoints`。
- 对 `加减法`、`应用题`、`句型`、`古诗` 等模糊 topic 有明确处理策略。

**预计工时**: 3 小时

---

### 任务 1.2: 新增题库校验脚本
**所属应用**: homework-tutor
**所属模块**: 工具脚本
**影响文件**:
- 新增: `scripts/validate-question-bank.js`

**描述**: 自动检查题库映射完整度，输出未归类题目和各学期题量分布。

**详细步骤**:
1. 加载 `questions.js`、`grade1.js` ~ `grade6.js` 与教材映射表。
2. 输出每个年级/学科/学期的题量统计。
3. 输出未命中教材映射或知识点为空的题目清单。

**验收标准**:
- 可以一键统计题库分布。
- 未映射题目会被明确列出。
- 脚本结果可作为人工复核依据。

**预计工时**: 1 小时

---

## 阶段二：题库装载与数据兼容（优先级: 高）

### 任务 2.1: 在 questions.js 中补齐题目元数据并构建学期索引
**所属应用**: homework-tutor
**所属模块**: 静态题库脚本
**影响文件**:
- 修改: `questions.js`

**描述**: 在不推翻现有 `QB[g][subject]` 结构的前提下，为每题自动补齐 `semester` 和 `knowledgePoints`。

**详细步骤**:
1. 新增 `annotateQuestionBank()` 统一遍历所有题目。
2. 按“题目级覆写优先、topic 映射其次”补齐标准字段。
3. 生成学期索引对象，供前端直接按学期取题。

**验收标准**:
- 现有 `QB[g][subject]` 访问方式仍可用。
- 每题都能拿到 `semesterLabel` 与 `knowledgePoints`。
- 学期索引为空时有明确提示信息。

**预计工时**: 2 小时

---

### 任务 2.2: 仅对必要题目补覆写
**所属应用**: homework-tutor
**所属模块**: 年级题库脚本
**影响文件**:
- 修改: `grade1.js`
- 修改: `grade2.js`
- 修改: `grade3.js`
- 修改: `grade4.js`
- 修改: `grade5.js`
- 修改: `grade6.js`

**描述**: 对无法仅靠 topic 精准归册的题目补充精确元数据或题目级标记。

**详细步骤**:
1. 基于校验脚本输出定位未命中或误命中的题目。
2. 对少量模糊题添加覆写信息，而不是整体重写所有题目。
3. 再次运行校验脚本确认无漏标。

**验收标准**:
- 未映射题目数为 0。
- 模糊题不会被错误分配到错误学期。
- 题库原始题干和现有难度体系保持不变。

**预计工时**: 3 小时

## 阶段三：前端学期筛题与展示（优先级: 高）

### 任务 3.1: 增加学期筛选状态与开战逻辑
**所属应用**: homework-tutor
**所属模块**: 前端单页
**影响文件**:
- 修改: `index.html`

**描述**: 在闯关入口引入学期选择，并按学期筛选题目。

**详细步骤**:
1. 在副本页增加学期选择 UI 或状态。
2. 修改 `startBattle()`，从学期索引中读取题池。
3. 根据产品决策调整通关进度 key 是否包含 semester。

**验收标准**:
- 用户可以明确选择上册或下册。
- 开战后只会抽到所选学期的题。
- 题池为空时不会进入异常状态。

**预计工时**: 2 小时

### 任务 3.2: 在题目与错题列表展示学期和知识点
**所属应用**: homework-tutor
**所属模块**: 前端单页
**影响文件**:
- 修改: `index.html`

**描述**: 把新增元数据显示到题目头部、错题库和相关摘要区。

**详细步骤**:
1. 题目头部显示 `学科 · 学期 · 主知识点`。
2. 错题库列表显示 `学期 + topic + 日期`。

**验收标准**:

- 缺少学期的题目仍可正常渲染。
- 错题库条目能直观看到所属册次。

**预计工时**: 1.5 小时
